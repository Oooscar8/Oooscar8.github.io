工程问题建模与仿真案例2-代码说明

520021911048 唐波昊

代码文件夹介绍

  • 代码文件夹中包含4个文件夹，1个readme文档。其中 basic 代表基础要求部分，expand1，expand2，expand3，expand4 代表拓展1，2，3，4部分。

basic文件夹

  • main.m 文件：基础任务主函数。
  • A.m 文件：用符合负指数分布随机数，随机生成切换器 A 的寿命和状态。
  • B.m 文件：用符合负指数分布随机数，随机生成切换器 B 的寿命和状态。
  • Node.m 文件：输入切换器 A 和 B 的状态，得到 A 和 B 所在节点的状态。
  • System_condition.m 文件：利用输入的所有 n 个节点的状态，得到系统当前的状态。
  • System_life.m 文件：将调用前面 4 个函数文件，对于输入的 n，随机生成一个系统，并得到这个随机系统的寿命。

expand1文件夹

  • KeYong.m 文件：生成可用性（拓展1任务主函数）。
  • P_keyong.m 文件：生成不同情况的概率。

expand2文件夹

  • time_bianbuchang.m 文件：生成变步长方法运行时间（拓展2任务主函数1）。
  • time_dengbuchang.m 文件：生成等步长方法运行时间（拓展2任务主函数2）。
  • A.m 文件：用符合负指数分布随机数，随机生成切换器 A 的寿命和状态。
  • B.m 文件：用符合负指数分布随机数，随机生成切换器 B 的寿命和状态。
  • Node.m 文件：输入切换器 A 和 B 的状态，得到 A 和 B 所在节点的状态。
  • System_condition.m 文件：利用输入的所有 n 个节点的状态，得到系统当前的状态。
  • System_life.m 文件：将调用前面 4 个函数文件，对于输入的 n，随机生成一个系统，并得到这个随机系统的寿命。

expand3文件夹

  • main.m 文件：拓展3主函数。
  • A.m 文件：用符合负指数分布随机数，随机生成切换器 A 的寿命和状态。
  • B.m 文件：用符合负指数分布随机数，随机生成切换器 B 的寿命和状态。
  • Node.m 文件：输入切换器 A 和 B 的状态，得到 A 和 B 所在节点的状态。
  • System_condition.m 文件：利用输入的所有 n 个节点的状态，得到系统当前的状态。
  • System_life.m 文件：将调用前面 4 个函数文件，对于输入的 n，随机生成一个系统，并得到这个随机系统的寿命。

expand4文件夹

  • newmain.m 文件：拓展4任务主函数。
  • newA.m 文件：用符合负指数分布随机数，随机生成切换器 A 的寿命和状态。
  • new’B.m 文件：用符合负指数分布随机数，随机生成切换器 B 的寿命和状态。
  • newNode.m 文件：输入切换器 A 和 B 的状态，得到 A 和 B 所在节点的状态。
  • newSystem_condition.m 文件：利用输入的所有 n 个节点的状态，得到系统当前的状态。
  • newSystem_life.m 文件：将调用前面 4 个函数文件，对于输入的 n，随机生成一个系统，并得到这个随机系统的寿命。
  • draw1.m 文件：绘制原始方案和改进方案的MTTF对比曲线。
  • draw2.m 文件：绘制原始方案和改进方案的系统可靠性对比曲线。
  • draw3.m 文件：绘制原始方案和改进方案的阻塞故障占比对比曲线。

运行使用说明

  • 基础任务 ：运行 main.m文件。
  • 拓展1 ：运行 KeYong.m 文件。
  • 拓展2 ：运行 time_bianbuchang.m，time_dengbuchang.m文件。
  • 拓展3 ：运行 main.m 文件。
  • 拓展4 ：运行newmain.m，draw1.m，draw2.m，draw3.m 文件。
